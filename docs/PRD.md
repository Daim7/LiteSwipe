# 轻滑相册 (LiteSwipe) — Android 相册应用 PRD

> **版本**: v1.1  
> **日期**: 2026-04-08  
> **架构**: MVVM (Kotlin + XML)  
> **最低 API**: Android 8.0 (API 26)

---

## 一、产品概述

轻滑相册 (LiteSwipe) 是一款 Android 相册应用，UI 风格参考最新 iOS 相册设计（简洁、圆角、毛玻璃、大间距），支持本地照片/视频浏览与外部存储介质（USB / SD 卡）媒体的检测、预览及导入。主界面采用与 **iOS「照片」应用类似的底部标签导航**；支持 **Motion Photo（Android Live Photos / MVIMG）** 标识与长按播放内嵌视频；查看器内集成 **ExoPlayer** 全屏视频播放与自定义底部控制条。应用采用 **边到边（Edge-to-edge）** 布局：状态栏与导航栏透明，内容延伸至系统栏后方并通过 **WindowInsets** 正确处理安全区域。支持跟随系统深色模式。

---

## 二、用户角色

| 角色 | 描述 |
|------|------|
| 普通用户 | 使用手机浏览、查看、管理本地照片和视频；通过「导入」标签访问外部存储媒体并加入待导入队列、执行导入或删除操作 |

---

## 三、功能模块总览

```
LiteSwipe (轻滑相册)
├── 主导航：MainActivity + BottomNavigationView（图库 / 导入 / 最近删除）
├── 模块 A：本地媒体浏览（图库标签）
│   ├── A.1 按日期分组展示
│   └── A.2 多选批量操作
├── 模块 B：大图/视频查看（本地）
├── 模块 C：外部存储检测（由「导入」标签承载外部媒体，图库页无 Tab 切换）
├── 模块 D：外部存储媒体浏览（导入标签页）
│   ├── D.1 常驻选择圈 + 待导入队列交互
│   └── D.2 按日期分组 + 捏合缩放网格
├── 模块 E：大图/视频查看（外部存储）
├── 模块 F：待导入集合（ImportQueueManager + BottomSheet）
├── 模块 G：导入执行
└── 模块 H：回收站（最近删除标签）
```

## 四、详细功能需求

### 主导航：底部导航（MainActivity）

**实现**: 主 Activity 使用 **`BottomNavigationView`**，不再在各 Fragment 内使用自定义 Tab 栏。

| 编号 | 需求描述 | 优先级 |
|------|---------|--------|
| NAV-1 | 三个底部标签：**图库**、**导入**、**最近删除**，交互与 iOS「照片」底部 Tab 风格一致 | P0 |
| NAV-2 | 「图库」对应本地相册网格（模块 A） | P0 |
| NAV-3 | 「导入」对应外部存储媒体浏览与管理（模块 D），承担原「本地 / 外部」中的外部媒体入口 | P0 |
| NAV-4 | 「最近删除」对应回收站（模块 H） | P0 |

### 模块 A：本地媒体浏览（图库标签）

**页面**: `GalleryFragment`（作为底部「图库」Tab 内容）

| 编号 | 需求描述 | 优先级 |
|------|---------|--------|
| A-1 | 以网格（Grid）形式展示本地相册中的所有图片和视频，**按日期分组**（同一天归为一组） | P0 |
| A-2 | 每个日期分组有标题头（如「2026年4月8日」「昨天」「今天」） | P0 |
| A-3 | 网格默认 3 列，图片等比裁剪为正方形缩略图，圆角展示 | P0 |
| A-4 | **双指缩放网格**：双指捏合/张开可调整网格列数（2列↔3列↔5列↔7列），类似 iOS 相册缩放效果 | P0 |
| A-5 | 视频缩略图左下角显示时长标签（如「0:30」） | P0 |
| A-6 | **Motion Photo（MVIMG）**：网格上显示 **LIVE** 角标；在查看器中**长按**可播放内嵌视频片段 | P0 |
| A-7 | 顶部显示应用标题「轻滑相册」，风格参考 iOS 相册的大标题导航栏（需适配边到边内边距） | P1 |
| A-8 | 支持滑动加载更多（分页加载，每页 50 张） | P1 |
| A-9 | 空状态提示：无媒体时展示空状态插画 + 文字 | P2 |
| A-10 | **多选模式**：长按进入多选 → 可勾选多张 → 底部出现「移入回收站 (N)」按钮 | P0 |
| A-11 | 多选模式下顶部显示「已选择 N 项」+ 「全选」/「取消」按钮 | P0 |
| A-12 | 多选模式下每个日期分组标题旁显示**「全选」按钮**，点击可选中该日期下所有图片 | P0 |

**交互**:
- 单击任意图片/视频 → 进入**大图/视频查看模式（模块 B）**
- 长按任意图片/视频 → 进入**多选模式**

**说明**: 图库页**不再**使用 TabLayout 或分段控件切换「本地 / 外部」；外部媒体统一由底部 **「导入」** 标签进入（见模块 C、D）。

### 模块 B：大图/视频查看（本地）

**页面**: `PhotoViewerActivity`

| 编号 | 需求描述 | 优先级 |
|------|---------|--------|
| B-1 | 图片：全屏展示，支持双指缩放 | P0 |
| B-2 | 视频：**ExoPlayer** 全屏播放，**自定义底部控制条**（播放/暂停、进度等） | P0 |
| B-3 | **左右滑动**：切换上一张 / 下一张（ViewPager 形式） | P0 |
| B-4 | **向上滑动**：将当前图片/视频移入回收站（直接执行，无确认弹窗） | P0 |
| B-5 | **向下滑动**：退出查看，返回图库网格 | P0 |
| B-6 | 移入回收站后，底部短暂显示「已移入回收站」+ 「撤销」链接（3 秒后消失） | P0 |
| B-7 | 底部半透明区域显示文件基本信息（文件名、日期、尺寸/时长） | P2 |
| B-8 | **Motion Photo**：支持 MVIMG；网格 LIVE 角标对应查看器内长按播放内嵌视频 | P0 |
| B-9 | 进入/退出时有平滑的共享元素过渡动画 | P1 |

**手势优先级规则**:
- 上下滑动（角度 > 60°）优先于左右滑动
- 左右滑动（角度 < 30°）触发翻页
- 中间角度（30°-60°）不响应，避免误操作
- 向上/下滑动需要滑动距离 > 100dp 才触发，有拖拽跟手反馈

**移入回收站流程**:
```
向上滑动超过阈值
  → 图片飞出顶部动画
  → 自动显示下一张（若为最后一张则返回图库）
  → 底部出现「已移入回收站 · 撤销」Snackbar（3秒）
  → 点击「撤销」可恢复
```

### 模块 C：外部存储检测与入口

**触发条件**: 系统检测到外部存储介质插入（USB OTG / SD 卡）

| 编号 | 需求描述 | 优先级 |
|------|---------|--------|
| C-1 | 通过 BroadcastReceiver 监听外部存储挂载/卸载事件 | P0 |
| C-2 | **图库（模块 A）页面不再展示 TabLayout / 分段选择器** 用于切换外部存储 | P0 |
| C-3 | 外部媒体的浏览、选择与导入入口统一为底部导航 **「导入」** 标签（模块 D） | P0 |
| C-4 | 无外部存储时，「导入」页给出相应空状态或提示；挂载后自动或可刷新列出外部媒体 | P0 |
| C-5 | 外部存储卸载时，导入页状态更新，避免操作已失效路径 | P0 |

**说明**: 原 PRD 中「首页顶部分段选择器：本地 / 外部」已移除；**本地与外部的信息架构**改为：**图库 = 仅本地**，**导入 = 外部（及待导入流程）**。

### 模块 D：外部存储媒体浏览（导入标签页）

**页面**: `ExternalGalleryFragment`（作为底部「导入」Tab 内容，**非**全屏独立栈顶跳转）

| 编号 | 需求描述 | 优先级 |
|------|---------|--------|
| D-1 | 扫描外部存储中所有图片和视频文件，按日期分组网格展示 | P0 |
| D-2 | **双指捏合缩放网格**：与图库一致，可调整列数（2↔3↔5↔7 列） | P0 |
| D-3 | **始终显示选择圆圈**：无需先进入「选择模式」；每张缩略图上的圆圈常驻 | P0 |
| D-4 | **点击圆圈**：将该项 **加入 / 移出** 全局待导入队列（待导入合集），状态与 `ImportQueueManager` 同步 | P0 |
| D-5 | **点击图片区域（非圆圈）**：进入外部存储全屏查看器（模块 E） | P0 |
| D-6 | 当有项目被选中时，底部固定显示 **「导入」** 与 **「删除」** 两个按钮 | P0 |
| D-7 | 顶部在有待导入项时显示 **待导入队列横条/摘要**（与 BottomSheet 队列联动） | P0 |
| D-8 | 顶部工具区：**「选择 / 完成」** 切换、**「全选」**、**「取消」** | P0 |
| D-9 | 每个日期分组标题旁可保留 **「全选」**（与全选/取消逻辑一致） | P1 |

**交互摘要**:
- 圆圈 ⇄ 待导入队列；图像 ⇄ 查看器；底部批量 **导入 / 删除**；顶部队列条 + 选择态切换。

### 模块 E：大图/视频查看（外部存储）

**页面**: `ExternalPhotoViewerActivity`

| 编号 | 需求描述 | 优先级 |
|------|---------|--------|
| E-1 | 图片：全屏展示，支持双指缩放；视频：**ExoPlayer** 播放 + **自定义底部控制条** | P0 |
| E-2 | **左右滑动**：切换上一张 / 下一张 | P0 |
| E-3 | **向上滑动**：隐藏当前项（从浏览列表中标记隐藏，不物理删除外部存储文件）；流程可含确认（与实现一致） | P0 |
| E-4 | **向下滑动**：仍将当前项 **加入待导入队列**（与导入页圆圈行为一致） | P0 |
| E-5 | 向下滑动加入队列后，有「已加入待导入」等动效或提示 | P1 |
| E-6 | **顶栏**：左侧 **返回**；居中 **「当前序号 / 总数」**（如 N/Total）；右侧 **选择圆圈**（点击切换该项是否在待导入队列中） | P0 |
| E-7 | **底栏**：**横向可滚动**的已选（或队列相关）**缩略图行** + **「导入」** 按钮；**不再**使用旧版 **「加入待导入」/「导入全部」** 的单一旧布局 | P0 |
| E-8 | 通过 BottomSheet（模块 F）可查看与管理完整待导入队列 | P0 |

**向下滑动加入队列流程**:
```
向下滑动超过阈值
  → 动效提示加入待导入
  → ImportQueueManager 更新
  → 自动显示下一张（或保持与实现一致的导航行为）
  → 顶栏/底栏与 BottomSheet 队列数量同步
```

**向上滑动隐藏流程**（与实现一致）:
```
向上滑动超过阈值 → 必要时确认「隐藏此图片？」
  → 确认：从浏览列表中移除（不删除外部文件），显示下一张
  → 取消：回弹原位
```

### 模块 F：待导入集合

**实现**: 全局单例 **`ImportQueueManager`**，队列 **持久化**，在页面与 Activity 切换后仍保留。

**呈现**: **BottomSheet** 对话框，可从 **导入页** 与 **外部查看器** 打开。

| 编号 | 需求描述 | 优先级 |
|------|---------|--------|
| F-1 | 使用 `ImportQueueManager` 作为唯一数据源，保证导入页、查看器、BottomSheet 状态一致 | P0 |
| F-2 | BottomSheet 内 **网格** 展示队列中所有项 | P0 |
| F-3 | **点击某格**：从队列中 **移除** 该项 | P0 |
| F-4 | **「导入全部」**：触发模块 G，导入队列内全部项 | P0 |
| F-5 | **「清空」**：清空待导入队列 | P0 |
| F-6 | 队列在 **Fragment / Activity 切换** 后仍然保持，直至用户导入、清空或逐项移除 | P0 |

**说明**: 原独立全屏 `ImportQueueFragment` 主导航方式可弱化或替换为 BottomSheet；若代码中仍保留 Fragment，其职责应与 Manager + Sheet 对齐。

- 原文件路径

## 五、UI/UX 设计规范

### 5.1 设计风格

| 项目 | 规格 |
|------|------|
| 设计参考 | iOS 18 Photos App（含底部 Tab 导航形态） |
| 主色调 | 系统蓝 #007AFF、纯白/纯黑背景、自适应文字颜色 |
| 深色模式 | 跟随系统自动切换，使用 Material You DayNight 主题 |
| 圆角 | 卡片 12dp、按钮 8dp、缩略图 4dp |
| 字体 | 标题 28sp Bold、正文 16sp、辅助 12sp |
| 间距 | 网格间距 2dp、页面边距 16dp |
| 动画 | 转场/手势 300ms ease-in-out |

### 5.2 边到边（Edge-to-edge）与系统栏

| 项目 | 规格 |
|------|------|
| 状态栏 | **透明**，内容可绘制到其后方 |
| 导航栏 | **透明**（或手势导航条区域协调一致） |
| 内容与安全区 | 使用 **WindowInsets**（如 `WindowInsetsCompat`）为工具栏、底部导航、BottomSheet 等预留 **padding / margin**，避免与状态栏、刘海、导航栏重叠 |
| 底部导航 | `BottomNavigationView` 需处理底部 inset，避免与系统手势条冲突 |

### 5.3 手势规格

| 手势 | 场景 | 触发方式 | 反馈 |
|------|------|---------|------|
| 左右翻页 | 大图查看 | 角度 < 30°，跟手滑动 | ViewPager 弹性翻页 |
| 向上操作 | 大图查看 | 角度 > 60° 且 dy < 0，> 100dp | 图片跟手上移 + 提示 |
| 向下操作 | 大图查看 | 角度 > 60° 且 dy > 0，> 100dp | 本地：退出；外部：加入待导入队列等 |
| 双指缩放 | 网格浏览 | 双指捏合/张开 | 网格列数平滑过渡（2↔3↔5↔7列） |
| 长按 | 查看器（Motion Photo） | 长按 | 播放 MVIMG 内嵌视频 |

**网格缩放规格**:
- 双指张开（放大）：减少列数，缩略图变大（7→5→3→2 列）
- 双指捏合（缩小）：增加列数，缩略图变小（2→3→5→7 列）
- 切换时有平滑缩放动画（200ms）
- 记住用户选择的列数偏好（持久化到 SharedPreferences）

## 九、已确认决策

| 序号 | 决策 |
|------|------|
| 1 | v1.1 支持图片和视频；视频在查看器中使用 ExoPlayer 与自定义底栏 |
| 2 | 照片按日期分组展示（类似 iOS 按天分组） |
| 3 | 主导航为三 Tab：**图库 / 导入 / 最近删除**，对齐 iOS 照片底部结构 |
| 4 | 外部媒体仅通过 **导入** Tab 进入；图库页 **无** TabLayout 切换外部存储 |
| 5 | 待导入队列由 **ImportQueueManager** 全局持久化；**BottomSheet** 统一编辑队列 |
| 6 | 导入页常驻选择圈；点圈改队列，点图进查看器；支持导入与删除批量操作 |
| 7 | 外部查看器顶栏：返回 + N/Total + 选择圈；底栏：横向缩略图 + 导入；下滑仍加入队列 |
| 8 | 支持 **Motion Photo（MVIMG）**：网格 LIVE 角标，查看器长按播内嵌视频 |
| 9 | 深色模式跟随系统 |
| 10 | 本地多选批量删除（移入回收站） |
| 11 | 导入 = 复制（保留外部存储原文件） |
| 12 | 本地删除 = 移入回收站（30 天后自动永久删除），无确认弹窗 + 可撤销 |
| 13 | 外部存储向上滑动 = 隐藏（仅从浏览列表隐藏，不物理删除） |

## 十一、v1.1 变更摘要（相对 v1.0）

1. **主导航**：各 Fragment 内自定义 Tab 改为 **`BottomNavigationView`**，三 Tab：**图库**、**导入**、**最近删除**。  
2. **模块 D**：导入为独立底栏 Tab；**常驻选择圈**；点圈维护队列、点图进查看器；底部 **导入/删除**；顶部队列条与 **选择·完成 / 全选 / 取消**；支持捏合缩放网格。  
3. **模块 E**：顶栏 **返回 | N/Total | 选择圈**；底栏 **横向缩略图 + 导入**；圈与下滑均可改队列；移除旧 **「加入待导入」/「导入全部」** 主导布局。  
4. **模块 F**：**`ImportQueueManager`** 单例持久化队列；**BottomSheet** 展示网格、点格移除、**导入全部** 与 **清空**；导入页与查看器均可打开。  
5. **模块 C**：图库 **不再** 使用 TabLayout/分段控件切外部存储；外部由 **导入** Tab 承担。  
6. **Motion Photo**：MVIMG **LIVE** 角标 + 查看器 **长按** 播内嵌视频。  
7. **视频**：查看器内 **ExoPlayer** 全屏播放与 **自定义底部控制条**。  
8. **系统 UI**：**边到边**，状态栏/导航栏透明，**Insets** 适配内容与安全区。
