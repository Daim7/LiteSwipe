## 三、关键问题与解决方案

### 3.1 TransactionTooLargeException (Activity 间传大量数据)
- **问题**: 打开查看器时传递数千个 `MediaItem` 导致 Binder 事务超限崩溃
- **解决**: 创建 `SharedMediaData` 单例，在启动 Activity 前存入数据，Activity 中取出，`onDestroy` 时清除

### 3.2 外部存储检测闪退
- **问题**: `StorageBroadcastReceiver` 在 AndroidManifest 中静态注册但构造函数需要参数，导致系统无法实例化
- **解决**: 移除 Manifest 中的静态注册，改为在 `MainActivity` 中动态注册。API 33+ 使用 `Context.RECEIVER_EXPORTED`

### 3.3 PhotoView 与 ViewPager2 触摸冲突
- **问题**: 放大图片后左右平移会误触发 ViewPager2 翻页
- **解决**: 创建 `PhotoViewContainer` 自定义 ViewGroup，在 `dispatchTouchEvent` 中检测多指或放大状态时调用 `requestDisallowInterceptTouchEvent(true)`

### 3.4 Scoped Storage 下的文件删除
- **问题**: Android 10+ 不允许应用直接删除 MediaStore 中的文件
- **解决**: 
  - 移入回收站: 复制文件到应用私有目录 `filesDir/trash/`，Room 记录元数据，主页查询时过滤
  - 永久删除: Android 11+ 使用 `MediaStore.createDeleteRequest()` 弹系统确认，之后清理 Room 记录和私有目录副本

### 3.5 动态照片检测
- **问题**: 不同厂商的 Motion Photo 格式不同（Google Camera 用 `Camera:MotionPhoto`, 小米用 `GCamera:MicroVideo`）
- **解决**: `MotionPhotoHelper` 解析 XMP 元数据，支持多种标签。首页使用文件名启发式检测（如 "MVIMG_" 前缀），查看器使用完整 XMP 解析

### 3.6 动态照片播放闪黑
- **问题**: 切换到 PlayerView 播放嵌入视频时第一帧会闪黑
- **解决**: 播放前立即设置 `PlayerView.visibility = VISIBLE` 并 `PhotoView.visibility = INVISIBLE`，消除切换间隙

### 3.7 视频控制条被底部信息栏遮挡
- **问题**: 全屏 ViewPager 导致 ExoPlayer 控制条位于信息栏之下
- **解决**: 图库查看器中 ViewPager 底部约束改为信息栏上方 (`app:layout_constraintBottom_toTopOf="@id/infoBar"`)

### 3.8 XML 布局 z-order 问题
- **问题**: ConstraintLayout 中后声明的 View 渲染在先声明的之上，导致浮动按钮被 ViewPager 遮盖
- **解决**: 将 ViewPager 放在 XML 最前面（最先声明 = 最底层），浮动 UI 元素放在后面

### 3.9 导入页点击行为错误
- **问题**: `isMultiSelectMode = true` 时 MediaAdapter 的整个 itemView 点击都触发选择，用户无法点击图片打开查看器
- **解决**: 在 MediaAdapter 添加 `separateCheckAndView` 参数，为 `true` 时 itemView 点击始终打开查看器，选择只由 checkIcon 的点击处理

### 3.10 状态栏遮挡内容
- **问题**: `enableEdgeToEdge()` 后内容延伸到状态栏区域，标题被遮挡
- **解决**: 通过 `ViewCompat.setOnApplyWindowInsetsListener` 给 `fragmentContainer` 添加状态栏高度的 top padding，给 `BottomNavigationView` 添加导航栏高度的 bottom padding

## 四、ADB 调试经验

### 4.1 设备连接问题
- 小米设备经常出现 `device offline` 或 `no devices/emulators found`
- 解决: `adb reconnect` + 等待 3-5 秒后重试
- 安装命令: `adb install -r -t "app\build\outputs\apk\debug\app-debug.apk"`
- `-r` 替换已有应用, `-t` 允许测试包

### 4.2 安装失败
- `INSTALL_FAILED_USER_RESTRICTED`: 小米特有问题，需在开发者选项中关闭"USB 安装监控"并开启"USB 调试(安全设置)"

## 五、代码架构关键点

### 5.1 Fragment 管理方式
- 使用 `show/hide` 模式而非 `replace` 进行 Tab 切换
- 好处: 保留 Fragment 视图状态，切换瞬间无重建开销
- 注意: `onHiddenChanged()` 代替 `onResume()` 感知可见性变化

### 5.2 全局状态管理
- `ImportQueueManager`: 单例管理待导入合集，使用 `MutableLiveData` 支持多页面观察
- `SharedMediaData`: 单例传递大量 MediaItem 列表（绕过 Binder 限制）
- `MainViewModel`: activityViewModels() 作用域，共享外部存储检测状态

### 5.3 MediaAdapter 多模式支持
- 标准模式: 点击打开查看器，长按进入多选
- 多选模式: 点击选择/取消，双击打开查看器
- 分离模式 (`separateCheckAndView`): 点击始终打开查看器，圆圈独立控制选择

## 六、后续开发建议

1. **性能优化**: 图片列表超过万张时考虑 Paging 3 分页加载
2. **国产ROM适配**: WorkManager 后台任务在小米/华为等设备上可能被省电策略杀掉，建议在 App 启动时额外做过期检测
3. **SAF 权限**: 外部存储删除操作可能在某些设备上需要 SAF 授权
4. **动态照片兼容性**: 不同厂商 Motion Photo 格式持续变化，需关注新设备的 XMP 标签
5. **视频播放**: 可考虑加入视频编辑（裁剪、旋转）等功能
6. **云同步**: 可接入 WebDAV 或其他云存储实现照片同步
